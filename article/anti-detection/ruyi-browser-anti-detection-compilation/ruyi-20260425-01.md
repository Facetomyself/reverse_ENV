##  一、整体架构与流程概览

云片验证码是一套前后端配合的行为验证系统，前端SDK名为  ** riddler-sdk  ** （版本  ` 0.2.2  ` ，构建版本  `
1.5.0(2021111001)  ` ），整个验证流程分为三步：

  *


    获取验证码(get) → 识别缺口 → 生成轨迹 + 加密参数 → 提交验证(verify)

通信方式为  ** JSONP  ** ，回调函数名固定前缀  ` ypjsonp  ` ，所有请求参数经过  ** AES + RSA 混合加密  **
后以 GET 方式提交。

##  三、身份伪装与指纹生成

###  3.1 浏览器指纹 (  ` fp  ` )

SDK使用  ** Fingerprint2  ** 库的  ` x64hash128  `
算法生成浏览器指纹，输出为32位十六进制字符串。在Python中我们通过  ` MD5(random + timestamp)  ` 模拟：

  *   *   *


    def generate_fingerprint() -> str:    seed = str(random.random()) + str(time.time())    return hashlib.md5(seed.encode()).hexdigest()

** 分析细节：  ** 原版Fingerprint2会收集数十项浏览器特征（User-
Agent、语言、屏幕分辨率、已安装插件、Canvas指纹、WebGL指纹等），将它们拼接后做hash。服务端可能会校验指纹的格式一致性，但不会校验是否对应真实浏览器，所以随机MD5即可通过。

###  3.2 设备追踪ID (  ` yp_riddler_id  ` )

SDK在Cookie中维护一个  ` yp_riddler_id  ` ，格式为标准UUID v4：

  *


    xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx

其中：

  * 第三段固定以  ` 4  ` 开头（表示UUID版本4）
  * 第四段首字符限定为  ` 8/9/a/b  ` （表示variant-1）

  *   *   *   *   *   *   *   *   *   *   *   *


    def generate_yp_riddler_id() -> str:    return "".join(        random.choice("0123456789abcdef") for _ in range(8)    ) + "-" + "".join(        random.choice("0123456789abcdef") for _ in range(4)    ) + "-4" + "".join(        random.choice("0123456789abcdef") for _ in range(3)    ) + "-" + random.choice("89ab") + "".join(        random.choice("0123456789abcdef") for _ in range(3)    ) + "-" + "".join(        random.choice("0123456789abcdef") for _ in range(12)    )

** 作用：  ** 这个ID在每次会话中保持不变，用于服务端追踪同一用户的多次验证行为。每次新建solver实例时重新生成一个，相当于模拟新用户。

###  3.3 JSONP回调名 (  ` cb  ` )

每次请求都生成一个11位随机字符串作为JSONP回调名，字符集为  ` a-z0-9  ` ：

  *   *   *   *   *


    def get_random_cb() -> str:    return get_random_str(11)   # 例如: "iecoitcebso"def get_random_str(length: int) -> str:    chars = string.ascii_lowercase + string.digits    return "".join(random.choice(chars) for _ in range(length))

** 分析：  ** SDK中对应  ` cbManager.preAdd()  `
方法，每次调用生成一个唯一名称并注册到全局回调管理器中。服务端会将响应包裹在  ` ypjsonp({...})  ` 中返回。

###  3.4 请求会话配置

  *   *   *   *   *   *   *   *   *


    self.session = requests.Session()self.session.headers.update({    "User-Agent": (        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) "        "AppleWebKit/537.36 (KHTML, like Gecko) "        "Chrome/120.0.0.0 Safari/537.36"    ),    "Referer": "https://www.yunpian.com/",})

** 关键点：  **

  * ` User-Agent  ` 必须是合法的浏览器UA，服务端会校验
  * ` Referer  ` 必须是  ` https://www.yunpian.com/  ` ，否则可能被拒绝
  * 使用  ` requests.Session()  ` 保持Cookie跨请求持久化


##  四、加密体系详解

云片验证码采用  ** AES + RSA 混合加密  ** 方案，这是一种常见的非对称+对称混合加密架构：

###  4.1 总体加密流程

  *   *   *   *   *   *   *   *


    原始payload(dict)    ↓ JSON.stringify (紧凑格式, 无空格)明文字符串    ↓ 生成随机AES key(16字节) + IV(16字节)    ↓    ├── AES-CBC-PKCS7加密明文 → Base64编码 → 参数 "i"    │    └── RSA-PKCS1v1.5加密(key+iv拼接, 32字节) → Base64编码 → 参数 "k"

最终提交的GET参数只有  ` i  ` (加密数据) 和  ` k  ` (加密密钥)，服务端用RSA私钥解出AES key和IV，再解密出原始JSON。

###  4.2 AES加密细节

  *   *   *   *   *   *   *


    def aes_encrypt(data: str, key: str, iv: str) -> str:    key_bytes = key.encode("utf-8")    # 16字节 → AES-128    iv_bytes = iv.encode("utf-8")      # 16字节    cipher = AES.new(key_bytes, AES.MODE_CBC, iv_bytes)    padded = pad(data.encode("utf-8"), AES.block_size)  # PKCS7填充    encrypted = cipher.encrypt(padded)    return base64.b64encode(encrypted).decode("utf-8")

** 对应SDK源码：  **

  *   *   *   *   *


    CryptoJS.AES.encrypt(    data,    CryptoJS.enc.Utf8.parse(key),    { iv: CryptoJS.enc.Utf8.parse(iv) })

  * ** 算法：  ** AES-128-CBC
  * ** 密钥长度：  ** 16字节（128位），由  ` get_random_str(16)  ` 生成，字符集  ` a-z0-9  `
  * ** IV长度：  ** 16字节，同样随机生成
  * ** 填充方式：  ** PKCS7（CryptoJS默认）
  * ** 输出编码：  ** Base64
  * ** 依赖库：  ** ` pycryptodome  ` (  ` from Crypto.Cipher import AES  ` )

###  4.3 RSA加密细节

  *   *   *   *   *


    def rsa_encrypt(data: str) -> str:    key = RSA.import_key(RSA_PUBLIC_KEY)    cipher = PKCS1_v1_5.new(key)    encrypted = cipher.encrypt(data.encode("utf-8"))    return base64.b64encode(encrypted).decode("utf-8")

** 对应SDK源码：  **

  *   *   *


    var encrypt = new JSEncrypt();encrypt.setPublicKey(rsaPublicKey);return encrypt.encrypt(aesKey + aesIv);

  * ** 算法：  ** RSA PKCS  #1  v1.5
  * ** 密钥长度：  ** 1024位（从公钥格式判断）
  * ** 加密内容：  ** AES密钥 + AES IV 直接拼接（32字节字符串）
  * ** 输出编码：  ** Base64
  * ** SDK使用的库：  ** JSEncrypt

###  4.4 完整加密函数

  *   *   *   *   *   *   *


    def encrypt_payload(payload: dict) -> dict:    data = json.dumps(payload, separators=(",", ":"))  # 紧凑JSON，无空格    aes_key = get_random_str(16)                       # 随机AES密钥    aes_iv = get_random_str(16)                        # 随机AES IV    encrypted_data = aes_encrypt(data, aes_key, aes_iv)    encrypted_key = rsa_encrypt(aes_key + aes_iv)      # key和iv拼接后RSA加密    return {"i": encrypted_data, "k": encrypted_key}

** 关键细节：  **

  * ` json.dumps  ` 使用  ` separators=(",", ":")  ` 生成紧凑格式，与JS的  ` JSON.stringify()  ` 输出一致
  * AES key 和 IV 是  ** 直接字符串拼接  ** 后整体RSA加密，不是分开加密
  * 每次请求都生成新的随机AES key和IV，所以即使同一份payload，每次加密结果也不同


##  五、JSONP通信协议

###  5.1 请求构造

所有API请求都走 GET 方式，参数附在URL上：

  *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *


    def _jsonp_request(self, path: str, payload: dict) -> dict:    # 1. 向payload注入公共字段    payload["fp"] = self.fingerprint              # 浏览器指纹    payload["address"] = "https://www.yunpian.com" # 来源页面    payload["yp_riddler_id"] = self.yp_riddler_id  # 追踪Cookie    # 2. 加密    encrypted = encrypt_payload(payload)    cb = get_random_cb()    # 3. 组装GET参数    params = {        "cb": cb,                     # JSONP回调名        "i": encrypted["i"],          # AES加密后的数据        "k": encrypted["k"],          # RSA加密后的AES密钥    }    if self.token:        params["token"] = self.token  # 验证令牌（首次get不需要）    params["captchaId"] = self.captcha_id    url = f"{HOST}/{VERSION}{path}"    resp = self.session.get(url, params=params)    # 4. 解析JSONP响应    match = re.search(r"ypjsonp\((.+)\)", resp.text)    return json.loads(match.group(1))

通过正则  ` ypjsonp\((.+)\)  ` 提取括号内的JSON字符串并解析。

  *   *   *   *   *   *   *   *   *   *   *


    def detect_gap_with_opencv(bg_url: str, front_url: str) -> float:    import cv2    import numpy as np    import httpx    # 下载并解码图片    bg_arr = np.frombuffer(httpx.get(bg_url).content, np.uint8)    front_arr = np.frombuffer(httpx.get(front_url).content, np.uint8)    bg_gray = cv2.imdecode(bg_arr, cv2.IMREAD_GRAYSCALE)    # 灰度版本    front_gray = cv2.imdecode(front_arr, cv2.IMREAD_GRAYSCALE)    bg_color = cv2.imdecode(bg_arr, cv2.IMREAD_COLOR)       # 彩色版本    front_color = cv2.imdecode(front_arr, cv2.IMREAD_COLOR)

** 策略1：Canny边缘 + 模板匹配（3组参数）  **

  *   *   *   *   *   *


        for low, high in [(50, 150), (100, 200), (80, 180)]:        bg_edge = cv2.Canny(bg_gray, low, high)        front_edge = cv2.Canny(front_gray, low, high)        res = cv2.matchTemplate(bg_edge, front_edge, cv2.TM_CCOEFF_NORMED)        _, max_val, _, max_loc = cv2.minMaxLoc(res)        candidates.append((max_loc[0], max_val, "canny"))

  * 使用3组不同的Canny阈值覆盖不同边缘强度场景
  * ` TM_CCOEFF_NORMED  ` 归一化互相关系数匹配
  * 每组产出一个候选结果：  ` (x坐标, 置信度, 策略名)  `

  *   *   *   *


        for method in [cv2.TM_CCOEFF_NORMED, cv2.TM_CCORR_NORMED]:        res = cv2.matchTemplate(bg_gray, front_gray, method)        _, max_val, _, max_loc = cv2.minMaxLoc(res)        candidates.append((max_loc[0], max_val, "gray"))

  * 跳过边缘提取，直接在灰度图上做模板匹配
  * 使用两种不同的相关度算法增加鲁棒性

** 策略3：彩色通道分别匹配  **

  *   *   *   *   *   *   *


       channel_xs = []    for c in range(3):  # B, G, R 三通道        res = cv2.matchTemplate(bg_color[:,:,c], front_color[:,:,c], cv2.TM_CCOEFF_NORMED)        _, max_val, _, max_loc = cv2.minMaxLoc(res)        channel_xs.append(max_loc[0])    avg_x = int(np.median(channel_xs))  # 取中位数    candidates.append((avg_x, 0.5, "color_ch"))

** 结果融合 — 过滤 + 聚类投票：  **

  *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *


        # 过滤：缺口不会在最左侧（滑块初始位置在left=0），一般在30%-90%范围    w = bg_gray.shape[1]    valid = [(x, s, m) for x, s, m in candidates if x > w * 0.25]    if not valid:        valid = candidates  # 全被过滤则不过滤    # 聚类投票：找最密集的x区域（窗口=总宽的5%）    xs = sorted([x for x, _, _ in valid])    best_cluster = []    for i, x in enumerate(xs):        cluster = [x2 for x2 in xs if abs(x2 - x) < w * 0.05]        if len(cluster) > len(best_cluster):            best_cluster = cluster    if best_cluster:        gap_x = int(np.median(best_cluster))   # 最大聚类的中位数    else:        gap_x = max(valid, key=lambda t: t[1])[0]  # fallback: 最高置信度    return gap_x / w

** 聚类逻辑详解：  **

  1. 将所有候选x坐标排序
  2. 以每个点为中心，统计距离在  ` 宽度*5%  ` （约24px）内的点数
  3. 选择聚集点数最多的簇
  4. 取该簇的中位数作为最终结果
  5. 这样做的好处：即使有1-2个策略给出错误结果，只要多数策略一致，就不会被误导

##  八、步骤3 — 轨迹生成与拟人化

###  8.1 轨迹格式

SDK要求的轨迹格式为二维数组：


    [[x1, y1, t1], [x2, y2, t2], ...]


  * ` x  ` : 页面上的  ** 绝对  ** x坐标（非相对滑动距离）
  * ` y  ` : 页面上的  ** 绝对  ** y坐标
  * ` t  ` : 相对于  ` mousedown  ` 事件的时间偏移（毫秒）

###  8.2 拟人化运动模型

  *   *   *   *   *   *   *


    def generate_human_track(distance_px: int, duration_ms: int = None) -> list:    if duration_ms is None:        duration_ms = random.randint(600, 1200)   # 总时长600-1200ms    base_x = random.randint(300, 600)  # 随机起始x坐标（模拟滑块在页面中的位置）    base_y = random.randint(400, 600)  # 随机起始y坐标    points = []    num_steps = random.randint(25, 45)  # 轨迹采样点数25-45个

** 缓出运动曲线 (Ease-Out)：  **

  *   *   *   *   *   *   *   *   *   *


        for i in range(num_steps):        progress = i / (num_steps - 1)    # 0.0 → 1.0        if progress < 0.7:            # 前70%时间：走完92%距离，使用2.5次幂缓出            eased = 1 - pow(1 - progress / 0.7, 2.5)            eased *= 0.92        else:            # 后30%时间：走完剩余8%距离（减速微调）            sub = (progress - 0.7) / 0.3            eased = 0.92 + sub * 0.08

** 运动曲线分析：  **

  * ` progress < 0.7  ` 阶段：  ` eased = (1 - (1 - t)^2.5) * 0.92  ` ，这是一条经典的缓出曲线，起始速度快，逐渐减速
  * ` progress >= 0.7  ` 阶段：线性走完剩余8%距离，模拟手指接近目标时的精细调整
  * 最终效果：前期快速移动，后期缓慢精准定位

** 过冲与回弹：  **

  *   *   *   *   *   *


            # 过冲：progress 85%-95% 时略微超过目标位置        if 0.85 < progress < 0.95:            eased += random.uniform(0.01, 0.03)  # 多滑1-3%        # 回弹：progress >= 95% 时轻微回拉        elif progress >= 0.95:            eased -= random.uniform(0, 0.01)      # 回退0-1%

** 物理含义：  **
真人操作时手指有惯性，接近目标会轻微超过再回来。过冲幅度1-3%（约5-15px），回弹幅度0-1%，非常细微但足以骗过简单的机器检测。

##  十三、总结

整个云片滑动验证码的安全架构可以概括为：

  1. ** 传输层安全  ** ：AES+RSA混合加密，防止参数被篡改和重放
  2. ** 行为层安全  ** ：分析鼠标轨迹特征判断是否为真人操作
  3. ** 身份层安全  ** ：浏览器指纹 + 设备追踪ID + UA校验

其中最薄弱的环节是  ** 行为层  **
——只要轨迹生成足够拟人化，服务端很难区分真人和程序。传输层的加密由于RSA公钥是硬编码在前端SDK中的，所以可以直接提取使用。身份层的各项参数都可以模拟生成。
