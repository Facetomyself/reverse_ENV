#  SOCKS5 单 Tab 单密码代理说明

本文档记录当前  ` C:\firefox\firefox  ` 中已经落地的 per-tab proxy 实现：ruyiPage 通过  `
--fpfile  ` 把  ` C:\firefox\fp.txt  ` 传给自定义 Firefox，Firefox 网络层读取代理列表，并让每个
Firefox container tab 绑定一条不同的 SOCKS5 密码代理。

所有示例均已脱敏，真实代理请只写在本机  ` C:\firefox\fp.txt  ` 。

##  1\. 架构

###  1.1 目标效果

最终效果：

  1. ` C:\firefox\fp.txt  ` 中开启代理轮换。

  2. ` fp.txt  ` 中按顺序列出多条 SOCKS5 密码代理。

  3. ruyiPage 启动 Firefox 时调用  ` FirefoxOptions.set_fpfile(...)  ` 。

  4. Firefox 进程收到启动参数  ` --fpfile <session fpfile> ` 。

  5. 测试脚本创建多个 Firefox container tab。

  6. 每个 container tab 的网络请求带有不同的  ` OriginAttributes.mUserContextId  ` 。

  7. ` nsProtocolProxyService::Resolve_Internal(...)  ` 根据  ` userContextId  ` 按顺序选代理。

  8. Firefox SOCKS 层用  ` nsProxyInfo  ` 里的 username/password 完成 SOCKS5 RFC1929 认证。

###  1.2 数据流

text


    C:\firefox\fp.txt
      |
      | ruyiPage 测试脚本复制为 session fpfile
      v
    <artifact>\profile\fp.txt
      |
      | FirefoxOptions.set_fpfile(...)
      v
    firefox.exe --fpfile <artifact>\profile\fp.txt
      |
      v
    nsProtocolProxyService::Resolve_Internal(channel, ...)
      |
      | StoragePrincipalHelper::GetOriginAttributesForNetworkState(...)
      v
    OriginAttributes.mUserContextId
      |
      | GetRuyiHTTPProxyFromFPFile(userContextId)
      v
    按 userContextId 分配 proxyIndex
      |
      v
    NewProxyInfo_Internal(type=socks, username, password, connectionIsolationKey)
      |
      v
    nsSOCKSIOLayer SOCKS5 username/password auth
      |
      v
    目标网站，如 https://browserscan.net/

###  1.3 为什么必须用 container tab

普通 tab 没有稳定的非零  ` userContextId  ` ，网络层无法可靠区分“第几个 tab”。

Firefox container tab 会带  ` OriginAttributes.mUserContextId  ` ，因此网络层可以按这个 ID
做绑定：

text


    第 1 个出现的非零 userContextId -> 代理列表第 1 条
    第 2 个出现的非零 userContextId -> 代理列表第 2 条
    第 3 个出现的非零 userContextId -> 代理列表第 3 条
    ...

绑定关系保存在 Firefox 进程内存中的  ` mRuyiHTTPProxyUserContexts  ` 。同一个 Firefox 进程内，同一个
container 后续请求继续使用同一个代理索引。

###  1.4 代理配置格式

推荐格式：

ini


    proxy.rotate.enabled=true
    proxy.rotate.exhausted=wrap
    proxy.rotate.proxy=socks5://proxy.example.com:1000:<USERNAME>:<PASSWORD-1>
    proxy.rotate.proxy=socks5://proxy.example.com:1000:<USERNAME>:<PASSWORD-2>
    proxy.rotate.proxy=socks5://proxy.example.com:1000:<USERNAME>:<PASSWORD-3>
    proxy.rotate.proxy=socks5://proxy.example.com:1000:<USERNAME>:<PASSWORD-4>
    proxy.rotate.proxy=socks5://proxy.example.com:1000:<USERNAME>:<PASSWORD-5>

兼容格式：

ini


    httpproxy.rotate.enabled=true
    httpproxy.rotate.exhausted=wrap
    httpproxy.rotate.proxy=host:port:username:password
    httpproxy.rotate.proxy=socks5://host:port:username:password
    socks5proxy.rotate.proxy=host:port:username:password

字段含义：

  * ` proxy.rotate.enabled=true  ` ：开启 per-tab 代理轮换。

  * ` proxy.rotate.exhausted=wrap  ` ：tab 数超过代理数时从头复用代理。

  * ` proxy.rotate.exhausted=direct|none|stop  ` ：代理不足时不继续分配轮换代理。

  * ` proxy.rotate.proxy=socks5://host:port:username:password  ` ：一条 SOCKS5 密码代理。

当前限制：

  * 单行代理格式按  ` host:port:username:password  ` 解析。

  * ` username  ` 和  ` password  ` 里不能包含冒号。

  * 当前解析不支持 IPv6 字面量 host。

###  1.5 SOCKS5 认证路径

SOCKS5 不走 HTTP  ` Proxy-Authorization  ` 头。

当前实现只需要在  ` nsProtocolProxyService  ` 创建  ` nsProxyInfo  ` 时写入：

text


    type = socks
    host = proxy host
    port = proxy port
    username = proxy username
    password = proxy password
    flags 包含 TRANSPARENT_PROXY_RESOLVES_HOST

后续 SOCKS5 握手由 Firefox 已有文件完成：

text


    C:\firefox\firefox\netwerk\socket\nsSOCKSIOLayer.cpp

如果  ` nsProxyInfo  ` 中有 username，SOCKS5 greeting 会提供 username/password auth 方法
` 0x02  ` ，再按 RFC1929 发送 username/password。

###  1.6 HTTP 密码代理兼容

per-tab SOCKS5 的核心在  ` nsProtocolProxyService.*  ` 。

同时保留了 HTTP 密码代理兼容：当代理类型是 HTTP，并且  ` nsProxyInfo  ` 中有 username/password 时，  `
nsHttpChannelAuthProvider.*  ` 会生成 Basic  ` Proxy-Authorization  ` 凭据。
