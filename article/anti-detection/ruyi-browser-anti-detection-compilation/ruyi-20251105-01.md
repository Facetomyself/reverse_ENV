###  ** |safari的内核编译环境构建  **

###  **
**

课程的浏览器开发包含chromium、gecko和webkit，分别是chrome、firefox和safari的内核，正在火热更新ing。。。
webkit内核小而精巧，是我个人最喜欢的。  以下所有内容均建立在win11虚拟机纯净环境基础上。

官方推荐使用chocolotey进行环境安装，这是一个windows平台下的包管理器。首先需要使用  管理员打开Powser Shell  ，输入以下命令：

  *


    Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

等待包管理器下载完毕之后，可以打开cmd，输入：

  *   *   *


    choco install -y xampp-81 python311 ruby git cmake gperf llvm ninjapython -m pip install pywin32git config --global core.autocrlf input

上边即是所有需要依赖的包。

包依赖安装完毕之后，开始拉取源码，源码需要能够访问github，这里的端口是本地的7890：

  *   *   *


    git config --global http.proxy http://127.0.0.1:7890git config --global https.proxy http://127.0.0.1:7890git clone https://github.com/webkit/webkit.git

然后进入漫长的源码拉取流程。

###  ** |编译流程
**

首先安装新版vs2022，把默认的C++桌面开发勾选，然后是两个单独的组件：

1.LLVM工具集的MSBuild支持

2.适用于windows的C++Clang

接下来切换到源码目录，设置环境变量：

  *   *


    set WEBKIT_LIBRARIES=%cd%\WebKitLibraries\winset WEBKIT_OUTPUTDIR=%cd%\WebKitBuild

然后先建一个bat脚本文件，再写入以下内容，代理写成自己的：

  *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *   *


    @echo offcd %~dp0path C:\xampp\apache\bin;%path%path C:\xampp\perl\bin;%path%path %ProgramFiles%\CMake\bin;%path%path %ProgramFiles(x86)%\Microsoft Visual Studio\Installer;%path%for /F "usebackq delims=" %%I in (`vswhere.exe -latest -property installationPath`) do set VSPATH=%%Irem set WEBKIT_LIBRARIES=%~dp0WebKitLibraries\winpath %~dp0WebKitLibraries\win\bin;%path%set WEBKIT_TESTFONTS=%~dp0Tools\WebKitTestRunner\fontsset DUMPRENDERTREE_TEMP=%TEMP%set CC=clang-clset CXX=clang-clrem set http_proxy=http://your-proxy:8080rem set https_proxy=%http_proxy%rem You can pass necessary JSC options https://github.com/WebKit/WebKit/blob/main/Source/JavaScriptCore/runtime/OptionsList.h#L83-L607.rem set JSC_dumpOptions=1rem set JSC_useJIT=0rem set JSC_useDFGJIT=0rem set JSC_useRegExpJIT=0rem set JSC_useDOMJIT=0rem You can show check TextureMapper FPS via WEBKIT_SHOW_FPS.rem set WEBKIT_SHOW_FPS=1call "%VSPATH%\VC\Auxiliary\Build\vcvars64.bat"cd %~dp0start powershell

写入完毕后运行。

然后就可以编译了：

  *


    perl Tools/Scripts/build-webkit --release

  *


    生成的浏览器在这里：WebKitBuild/Release/bin64/MiniBrowser.exe

用vs进行调试，那么需要：

  *   *


    perl Tools/Scripts/build-webkit --release --no-ninja --generate-project-onlydevenv WebKitBuild\Release\WebKit.sln

然后就可以在vs里边进行编译调试了。
