###  ** |  141.0.7390.37  版  本chromium源码拉取  **

###  **
**

为了保持环境一致，便于bug排查，我选择在虚拟机中进行教学：

  * VMware虚拟机软件

  * win11操作系统

  * 内存分配32G（可增加）

  * 硬盘300G（可增加）

  * 核心8核（可增加）

  * 禁止系统自动更新

需要安装：

  * •  ** Visual Studio 2022社区版（版本 >= 17.0.0）  **
  * •  ** 安装时必须勾选以下组件：  **
    * •  ** “使用 C++ 的桌面开发”  ** （英文：Desktop development with C++）
    * •  ** “MFC 和 ATL 支持”  ** （英文：VC++ MFC and ATL support，是一个子选项）


###  ** Windows 11 SDK（版本要正确！）  **

Chromium 需要特定版本的 Windows SDK：

> ** Windows 11 SDK，版本号：10.0.26100.4654  **

你可以：

  * •  在  ** Visual Studio Installer  ** 中勾选安装该版本的 SDK；
  * •  或者单独去微软官网下载安装；

** 还必须在SDK安装对应的 Debugging Tools（调试工具）  ** ，用于读取 Chromium 使用的大体积 PDB 调试符号文件：

  * •  ** Windows SDK Debugging Tools，版本 >= 10.0.26100.3323  **

如果你的 SDK 安装里没有调试工具，可以去：

> 控制面板 → 程序和功能 → 找到 “Windows Software Development Kit” → 右键「更改」→ 勾选「Debugging
> Tools for Windows」


###  ** Git（代码版本管理工具）  **

Chromium 使用 Git 管理代码，你需要：

  * •  去  https://git-scm.com/download/win  下载  ** Git for Windows  ** 并安装；
  * •  推荐使用默认选项；
  * •  安装后，确保你电脑上有  ` git  ` 命令可以在 CMD 或 PowerShell 中运行；

检查是否安装成功：


          *


    git version


###  depot_tools（Chromium 的代码同步与构建工具集）

这是 Chromium 团队开发的一套工具，用来下载代码、管理依赖、调用编译工具等。

####  安装步骤：

  1. 1\.  找一个目录，比如：  ` C:\src\depot_tools  `
  2. 2\.  打开 CMD，运行：

  *   *


    cd C:\srcgit clone https://chromium.googlesource.com/chromium/tools/depot_tools.git

  1. 将 depot_tools 添加到系统 PATH 环境变量的最前面

  * •  打开：控制面板 → 系统 → 高级系统设置 → 环境变量
  * •  找到  ` Path  ` ，把  ` C:\src\depot_tools  ` ** 放在最前面  ** （比其他 Python 或系统路径优先级高！）
  * •  同时，添加一个环境变量：

  *


    set DEPOT_TOOLS_WIN_TOOLCHAIN=0

  1. VS2022 安装在哪，这个位置以你自己安装的为准，比如：

  *


    set vs2022_install="C:\Program Files\Microsoft Visual Studio\2022\Community"

##  二、获取 Chromium 源代码

###  1\.  ** 配置 Git（全局设置，只需一次）  **


    cmd

    复制


    git config--global user.name "你的名字"
    git config--global user.email "你的邮箱@example.com"
    git config--global core.autocrlf false
    git config--global core.filemode false
    git config--global core.preloadindex true
    git config--global core.fscache true
    git config--global branch.autosetuprebase always

    git config --global core.longpaths true


这里需要配置好科学上网的工具，我使用clash：


    cmd

    复制


    git config--global http.proxy http://127.0.0.1:7890
    git config--global https.proxy http://127.0.0.1:7890


###  2\.  ** 创建一个目录并进入（比如叫 chromium）  **


    cmd

    复制


    mkdir chromium
    cd chromium


###  3\.  ** 使用 depot_tools 下载指定版本源码  **


    cmd

    复制


    gclient config https://chromium.googlesource.com/chromium/src.git

    gclient sync --revision src@ 141.0.7390.37 --with_tags --with_branch_heads


这个过程  ** 非常耗时  ** ！即使是高速网络也可能需要  ** 1小时以上，慢速网络可能要几个小时  ** 。 注意：

  * •  下载过程中不要让电脑休眠或断网；
  * •  如果出错，可以重新运行  ` gclient sync上边这一串代码  ` ；


###  4\.  ** 进入代码目录  **


    cmd

    复制


    cd src


到这里，你已经成功下载了 Chromium 的源代码


##  三、配置编译环境（GN + Ninja）

Chromium 使用  ** GN  ** 工具生成构建配置，再用  ** Ninja  ** （超快构建工具）来编译。


###  1\.  ** 生成构建配置（第一次运行）  **


    cmd

    复制


    gn gen out\Default


  * •  这会在  ` src  ` 目录下生成一个  ` out\Default  ` 文件夹，里面是构建配置；
  * •  你可以改成别的名字，比如  ` out\Release  ` 或  ` out\Debug_x64  ` ，但必须放在 `out`下；

你可以加一些编译选项，比如：


    cmd

    复制


    gn gen out\Default --args="is_debug=true is_component_build=true"


##  四、开始编译 Chromium

###  1\.  ** 编译 Chrome 主程序  **


    cmd

    复制


    autoninja -C out\Default chrome


  * •  这是推荐的编译方式，  ` autoninja  ` 是 depot_tools 提供的封装，会优化参数；
  * •  ` -C out\Default  ` 指定使用哪个构建目录；
  * •  编译时间可能很长，建议至少  ** 几个小时  ** ，取决于你的 CPU 和磁盘速度；

加快编译：

  * •  把  ` out\Default  ` 文件夹放到  ** SSD 硬盘上  **
  * •  关闭杀毒软件（比如 Windows Defender）对代码目录的扫描
  * •  使用多核 CPU（官方建议越多越好 ）


##  五、运行你编译出来的 Chromium

编译成功后，直接运行：


    cmd

    复制


    out\Default\chrome.exe


你已经成功编译并运行了自己版本的 Chromium


##  补充：用 Visual Studio 打开代码并调试（重要）

如果你想用  ** Visual Studio 打开代码并调试  ** ，在gn那一步可以：


    cmd

    复制


    gn gen --ide=vs --ninja-executable=autoninja out\Default
    devenv out\Default\all.sln


但注意：生成的  ` .sln  ` 工程文件包含几千个项目，  ** 加载会很慢  ** ，建议加过滤：


    cmd

    复制


    gn gen --ide=vs --ninja-executable=autoninja --filters=//chrome --no-deps out\Default

以上。
