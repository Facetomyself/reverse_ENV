#  Firefox 任务栏窗口图标编号说明

本文档说明这次“Firefox 多窗口任务栏图标编号”改动的目标、实现方式、涉及文件、每个文件的修改内容、验证方式和排查要点。

##  目标

在 Windows 上打开多个普通 Firefox 浏览器窗口时：

  * 任务栏上每个 Firefox 窗口显示为独立图标。
  * 每个图标带一个递增编号：第一个窗口是 1，第二个窗口是 2，第三个窗口是 3。
  * 新窗口编号只递增，不因为关闭旧窗口而重新编号。
  * 数字显示在 Firefox 原始图标的下边居中位置。
  * 保留 Firefox 原始图标本身，不用空白图标替代。
  * 跳过 popup 窗口和 taskbar tab 窗口。

##  核心结论

Windows 的 taskbar overlay icon 位置由系统固定，Firefox 侧不能把 overlay 从右下角移动到底部居中。

因此最终实现没有继续使用  ` setOverlayIcon()  ` ，而是改为：

  1. 给每个浏览器窗口设置独立 AUMID，让任务栏拆成多个 Firefox 图标。
  2. 生成一个新的窗口主图标：先绘制 Firefox 原始图标，再把红底白字编号画到图标下边居中。
  3. 通过  ` nsIWindowsUIUtils.setWindowIcon(win, smallIcon, bigIcon)  ` 把这个合成图标设置到对应窗口。

##  文件 1：browser/base/content/browser-init.js

###  修改目的

让 Firefox 浏览器窗口启动时，在 Windows 平台初始化任务栏图标编号服务。

###  修改内容

在浏览器初始化流程里增加 Windows 平台判断：


    if (AppConstants.platform == "win") {
      let { WindowsTaskbarWindowBadge } = ChromeUtils.importESModule(
        "resource:///modules/WindowsTaskbarWindowBadge.sys.mjs"
      );
      WindowsTaskbarWindowBadge.init();
    }


###  作用

  * 只在 Windows 上加载编号模块。
  * 避免 Linux、macOS 或其他平台加载 Windows-only 服务。
  * 每个 browser window 初始化时都会调用  ` init()  ` ，但模块内部有  ` #initialized  ` 防重复注册，所以不会重复安装回调。

##  文件 2：browser/modules/moz.build

###  修改目的

把新模块加入 Firefox 打包系统，并标记它属于 Windows 任务栏相关代码。

###  修改内容

新增文件归属声明：


    with Files("WindowsTaskbarWindowBadge.sys.mjs"):
        BUG_COMPONENT = ("Core", "Widget: Win32")
        SCHEDULES.exclusive = ["windows"]


在 Windows-only 模块列表中加入：


    "WindowsTaskbarWindowBadge.sys.mjs",


###  作用

  * ` ./mach build faster  ` 会把  ` WindowsTaskbarWindowBadge.sys.mjs  ` 复制到 dist 目录。
  * 运行时可以通过  ` resource:///modules/WindowsTaskbarWindowBadge.sys.mjs  ` 正常导入。
  * 文件归属到  ` Core :: Widget: Win32  ` ，符合它调用 Windows taskbar 和 Windows window icon API 的性质。

##  文件 3：browser/modules/WindowsTaskbarWindowBadge.sys.mjs

这是本功能的核心实现文件。

###  新增模块导出

文件导出：


    export const WindowsTaskbarWindowBadge = new WindowsTaskbarWindowBadgeService();


外部只需要调用：


    WindowsTaskbarWindowBadge.init();


###  依赖模块和服务

使用的模块：

  * ` AppConstants  ` ：判断平台是否为 Windows。
  * ` XPCOMUtils  ` ：延迟获取 XPCOM 服务。
  * ` EveryWindow  ` ：监听 Firefox browser window 的打开和关闭。
  * ` PrivateBrowsingUtils  ` ：判断窗口是否为隐私窗口。

使用的 Windows 服务：

  * ` nsIWinTaskbar  ` ：设置窗口的 taskbar group id。
  * ` nsIWindowsUIUtils  ` ：设置窗口大小图标。

服务声明：


    XPCOMUtils.defineLazyServiceGetters(lazy, {
      WindowsUIUtils: ["@mozilla.org/windows-ui-utils;1", Ci.nsIWindowsUIUtils],
      WinTaskbar: ["@mozilla.org/windows-taskbar;1", Ci.nsIWinTaskbar],
    });


###  初始化逻辑

` init()  ` 做两件事：

  1. 检查服务是否已经初始化。
  2. 检查当前平台和 taskbar 服务是否可用。

通过后调用：


    lazy.EveryWindow.registerCallback(...)


窗口打开时执行  ` #addWindow  (win)  ` ，窗口关闭时执行  ` #clearWindow  (win)  ` 。

###  防重复初始化

模块里有：


    #initialized = false;


` init()  ` 如果发现已经初始化，会直接返回。这保证  ` browser-init.js  ` 多次调用时不会重复注册 EveryWindow
callback。

###  哪些窗口会被编号

` #shouldBadgeWindow  (win)  ` 会过滤窗口：


    return (
      win &&
      !win.closed &&
      win.toolbar?.visible &&
      !win.document?.documentElement?.hasAttribute("taskbartab")
    );


因此会编号的是普通 browser window。

会跳过的是：

  * 已关闭窗口。
  * 没有普通 toolbar 的 popup 窗口。
  * 带  ` taskbartab  ` 属性的 taskbar tab 窗口。

###  编号规则

模块里有：


    #nextNumber = 1;
    #numbers = new WeakMap();


逻辑是：

  1. 如果窗口还没有编号，就取当前  ` #nextNumber  ` 。
  2. 写入  ` #numbers  ` 。
  3. ` #nextNumber  ++  ` 。

这表示编号是单调递增的：

  * 第一个普通窗口是 1。
  * 第二个普通窗口是 2。
  * 第三个普通窗口是 3。
  * 关闭 2 以后再打开新窗口，新窗口是 4。
  * 已存在窗口不会重新编号。

###  让任务栏拆成多个 Firefox 图标

只改窗口图标还不够。Windows 默认可能把同一应用窗口合并到一个任务栏图标里。

因此每个窗口都会设置独立 group id：


    let groupId = lazy.PrivateBrowsingUtils.isWindowPrivate(win)
      ? taskbar.defaultPrivateGroupId
      : taskbar.defaultGroupId;
    taskbar.setGroupIdForWindow(win, `${groupId}.window-${number}`);


普通窗口使用  ` defaultGroupId  ` ，隐私窗口使用  ` defaultPrivateGroupId  ` 。

最终 group id 类似：

  * ` Mozilla.Firefox.window-1  `
  * ` Mozilla.Firefox.window-2  `
  * ` Mozilla.Firefox.Private.window-3  `

这样 Windows taskbar 会把不同编号窗口显示为不同按钮。

###  为什么不用 taskbar overlay icon

最早验证过  ` getOverlayIconController(...).setOverlayIcon(...)  ` ，它能在任务栏图标上显示数字，但
Windows 把 overlay 固定放在右下角。

用户要求“数字弄在图标的下边居中”，所以不能继续用 overlay。

最终改为修改窗口主图标：


    this.#getWindowsUIUtils().setWindowIcon(win, image, image);


这里 small icon 和 big icon 都传同一个合成后的  ` imgIContainer  ` 。

###  图标生成方式

图标大小固定为：


    const ICON_SIZE = 64;


生成流程在  ` #paintBadgeIcon  (win, label)  ` ：

  1. 创建 64x64 canvas。
  2. 加载 Firefox 原始图标：


    image.src = "chrome://branding/content/icon64.png";


  3. 把原始图标画到 canvas：


    context.drawImage(image, 0, 0, ICON_SIZE, ICON_SIZE);


  4. 在底部居中画红色圆角矩形：


    let badgeWidth = label.length > 1 ? 34 : 28;
    let badgeHeight = 20;
    let badgeX = (ICON_SIZE - badgeWidth) / 2;
    let badgeY = 42;
    this.#fillRoundedRect(context, badgeX, badgeY, badgeWidth, badgeHeight, 10);


  5. 在红色矩形里画白色数字：


    context.fillStyle = "#fff";
    context.font = `700 ${label.length > 1 ? 15 : 20}px Arial, sans-serif`;
    context.textAlign = "center";
    context.textBaseline = "middle";
    context.fillText(label, ICON_SIZE / 2, badgeY + badgeHeight / 2 + 1);


  6. 把 canvas 转成 PNG data URL：


    return canvas.toDataURL("image/png");


  7. 再用  ` ChromeUtils.fetchDecodedImage()  ` 解码成  ` imgIContainer  ` ：


    let uri = Services.io.newURI(dataURL);
    let channel = Services.io.newChannelFromURI(...);
    return ChromeUtils.fetchDecodedImage(uri, channel);


###  为什么用 canvas，不用 SVG 内嵌 chrome 图片

曾经尝试用 SVG：


    <image href="chrome://branding/content/icon64.png" ... />


结果任务栏里只显示数字，Firefox 原始图标消失。

原因是 data SVG 解码时，内部引用的  ` chrome://branding/...  ` 图片没有可靠进入最终 Windows icon。

后来又尝试把 Firefox 图标转成 data URL 嵌到 SVG，任务栏回退显示了 Firefox 原图标，但数字没有稳定出现。

最终使用 canvas 直接生成自包含 PNG。这是目前验证通过的方式：

  * Firefox 原始图标保留。
  * 数字在底部居中。
  * ` setWindowIcon()  ` 拿到的是已经合成完成的 PNG，不需要 Windows 或 image decoder 再解析 SVG 内部依赖。

###  数字显示上限

模块里有：


    const MAX_DISPLAY_NUMBER = 99;


超过 99 的窗口显示为：


    99+


缓存 key 使用：


    let cacheKey = number > MAX_DISPLAY_NUMBER ? "overflow" : displayNumber;


因此 100、101、102 等窗口共用同一个  ` 99+  ` 图标。

###  图标缓存

模块使用：


    #imageCache = new Map();


相同编号只生成一次图标，后续直接复用 Promise。

这样避免同一个编号窗口重复生成 canvas、重复解码图片。

###  关闭窗口时的处理

关闭窗口时调用：


    #clearWindow(win) {
      this.#numbers.delete(win);
    }


只删除窗口对应编号记录，不减少  ` #nextNumber  ` ，也不重新编号其他窗口。

###  测试用注入点

模块提供几个  ` testOnly*  ` 方法：

  * ` testOnlySetTaskbar(taskbar)  `
  * ` testOnlySetWindowsUIUtils(windowsUIUtils)  `
  * ` testOnlySetImageLoader(loader)  `
  * ` testOnlySetIconPainter(painter)  `
  * ` testOnlyAddWindow(win)  `
  * ` testOnlyReset()  `

这些只供 browser test 使用，用于替换 Windows-only 服务、替换图标绘制和图片解码，避免测试依赖真实 Windows
taskbar。

##  文件 5：browser/modules/test/browser/browser_WindowsTaskbarWindowBadge.js

这是新加的 browser test 文件。

###  测试目标

测试两个主要行为：

  1. 普通 browser window 会得到单调递增编号，并设置独立 taskbar group id 和自定义窗口图标。
  2. popup/taskbar tab 这类非普通浏览器窗口不会被编号。

###  taskbar stub

测试里用  ` createTaskbarStub()  ` 模拟  ` nsIWinTaskbar  ` 。

它提供：

  * ` available: true  `
  * ` defaultGroupId  `
  * ` defaultPrivateGroupId  `
  * ` setGroupIdForWindow(win, groupId)  `

并用  ` records  ` 记录每个窗口收到的 group id。

###  WindowsUIUtils stub

测试里用  ` createWindowsUIUtilsStub(records)  ` 模拟  ` nsIWindowsUIUtils  ` 。

它记录：


    setWindowIcon(win, smallIcon, bigIcon)


这样测试可以确认模块确实调用了  ` setWindowIcon()  ` ，而不是旧的  ` setOverlayIcon()  ` 。

###  图标绘制和解码 stub

测试通过：


    TaskbarBadge.testOnlySetIconPainter(...)
    TaskbarBadge.testOnlySetImageLoader(...)


模拟图标绘制和解码。

测试断言最终用于解码的是：


    data:image/png;base64,...


这能防止回退到不稳定的 SVG 路径。

###  单调递增测试

测试名：


    test_window_numbers_are_monotonic


覆盖流程：

  1. 初始化模块。
  2. 当前主窗口得到编号 1。
  3. 打开两个新窗口，得到编号 2 和 3。
  4. 关闭编号 2 的窗口。
  5. 再打开一个新窗口，得到编号 4。
  6. 确认编号 3 的窗口没有被重新编号。

关键断言包括：

  * ` Mozilla.Firefox.window-1  `
  * ` Mozilla.Firefox.window-2  `
  * ` Mozilla.Firefox.window-3  `
  * ` Mozilla.Firefox.window-4  `

并断言 small icon / big icon 都带有：

  * 正确 number。
  * 正确 label。
  * ` bottom-center  ` 位置标记。
  * ` data:image/png;base64,  ` 图标 URI。

###  跳过非 browser window 测试

测试名：


    test_skips_non_browser_taskbar_windows


覆盖：

  * ` toolbar.visible === false  ` 的 popup 类窗口。
  * documentElement 带  ` taskbartab  ` 属性的窗口。

断言：


    Assert.equal(taskbar.records.size, 0)


表示这些窗口不会设置 taskbar group id，也不会设置编号图标。

##  运行时完整流程

普通 Firefox 窗口打开后，流程如下：

  1. ` browser-init.js  ` 在 Windows 上导入并初始化  ` WindowsTaskbarWindowBadge  ` 。
  2. ` WindowsTaskbarWindowBadge.init()  ` 注册 EveryWindow callback。
  3. EveryWindow 发现普通 browser window，调用  ` #addWindow  (win)  ` 。
  4. ` #addWindow  ()  ` 给窗口分配递增编号。
  5. 通过  ` nsIWinTaskbar.setGroupIdForWindow()  ` 设置唯一 group id。
  6. 通过 canvas 生成带编号的 Firefox PNG 图标。
  7. 通过  ` ChromeUtils.fetchDecodedImage()  ` 解码成  ` imgIContainer  ` 。
  8. 通过  ` nsIWindowsUIUtils.setWindowIcon(win, image, image)  ` 设置窗口图标。
  9. Windows 任务栏显示多个 Firefox 图标，每个图标下边居中有对应数字。

##  本功能最终效果

在 Windows 任务栏中，多个 Firefox 普通浏览器窗口会显示为多个独立 Firefox 图标。每个图标保留 Firefox
原始图案，并在图标底部居中显示红底白字编号，例如 1、2、3。
