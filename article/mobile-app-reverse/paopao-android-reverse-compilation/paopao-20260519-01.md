##  一、为什么需要系统级 Hook

###  1.1 从「定向 Hook」到「全景监控」

前面几篇的 Hook 都是「定向」的——你已经通过 jadx 找到了目标类和方法，然后写脚本去拦截。但这有一个前提： **你得知道 Hook 什么** 。

面对一个完全陌生的、重度混淆的 App，你常常连"登录逻辑在哪个 Activity、Token 存在哪个 SP key、H5 页面通过什么 JSBridge
通信"都不知道。这些信息不在 App 的业务代码中，而在 **Android Framework 的系统 API**
中流转——不管业务代码怎么混淆，它总得调用系统 API 来创建 Activity、发送 Intent、读写 SharedPreferences。Hook
这些系统 API，就像在操作系统层面安装了监控摄像头。

###  1.2 本篇覆盖的系统 API

系统组件  |  Hook 目标  |  你能看到什么
---|---|---
**Activity** |  生命周期方法  |  页面跳转路径、当前显示的页面
**Intent** |  构造与发送  |  页面间传递的数据（用户ID、Token、URL）
**BroadcastReceiver** |  接收侧分发漏斗  |  谁发的广播、谁收的、带了什么数据
**SharedPreferences** |  getString/putString 等  |  本地存储的配置、Token、开关
**ContentProvider** |  query/insert/update/delete  |  数据库操作、跨进程数据共享
**WebView** |  loadUrl/evaluateJavascript  |  H5 页面 URL、JS 注入、JSBridge 调用
**Clipboard** |  getText/setText  |  复制粘贴的内容（可能含敏感数据）
**Toast/Dialog** |  show/makeText  |  App 给用户的提示信息
**PackageManager** |  getInstalledPackages 等  |  App 检查了哪些其他 App（环境检测）

##  二、Hook Activity 生命周期：追踪页面跳转

###  2.1 Activity 生命周期回顾

每个 Android 页面都是一个 Activity。Activity 有一套标准的生命周期方法：


    onCreate → onStart → onResume → [用户可见]
                                        ↓
                                   onPause → onStop → onDestroy


但单看一个 Activity 的生命周期没用——分析师真正想知道的是"从 A 切到 B 时各方法的交错顺序"，因为这决定了"上一页"和"当前页"分别该
hook 在哪里：

![Activity A → B
Activity A → B 切换时的生命周期时序

记住这张图的三个判断： **onResume 标记当前页 / onCreate 拿跳转参数 / onPause + onResume 配对画页面流转图**
。下面的脚本就是按这个思路写的。

###  2.2 追踪所有 Activity 的生命周期


    // activity_tracker.js
    // 追踪所有 Activity 的创建、显示和销毁

    Java.perform(function() {
        var Activity = Java.use("android.app.Activity");

        // ====== onCreate：Activity 被创建 ======
        Activity.onCreate.overload("android.os.Bundle").implementation = function(savedInstanceState) {
            var activityName = this.getClass().getName();
            var shortName = activityName.split(".").pop();

            console.log("\n┌─ [Activity] " + shortName + " 创建");

            // 获取启动这个 Activity 的 Intent
            var intent = this.getIntent();
            if (intent !== null) {
                var action = intent.getAction();
                var data = intent.getData();
                var extras = intent.getExtras();

                if (action) console.log("│  Action: " + action);
                if (data) console.log("│  Data URI: " + data.toString());

                // 打印 Intent 携带的 Extras
                if (extras !== null) {
                    var keys = extras.keySet();
                    var iter = keys.iterator();
                    while (iter.hasNext()) {
                        var key = iter.next().toString();
                        var value = extras.get(key);
                        if (value !== null) {
                            var valueStr = value.toString();
                            if (valueStr.length > 200) {
                                valueStr = valueStr.substring(0, 200) + "...";
                            }
                            console.log("│  Extra: " + key + " = " + valueStr);
                        }
                    }
                }
            }

            console.log("│  完整类名: " + activityName);
            console.log("└─");

            this.onCreate(savedInstanceState);
        };

        // ====== onResume：Activity 变为可见（到前台）======
        Activity.onResume.implementation = function() {
            var shortName = this.getClass().getName().split(".").pop();
            console.log("[→ 前台] " + shortName);
            this.onResume();
        };

        // ====== onPause：Activity 暂停（离开前台）======
        Activity.onPause.implementation = function() {
            var shortName = this.getClass().getName().split(".").pop();
            console.log("[← 后台] " + shortName);
            this.onPause();
        };

        // ====== onDestroy：Activity 被销毁 ======
        Activity.onDestroy.implementation = function() {
            var shortName = this.getClass().getName().split(".").pop();
            console.log("[× 销毁] " + shortName);
            this.onDestroy();
        };

        console.log("[OK] Activity 生命周期追踪");
    });


运行效果：

    │  完整类名: com.example.app.ui.SplashActivity
    └─
    [→ 前台] SplashActivity
    [← 后台] SplashActivity
    [× 销毁] SplashActivity

    ┌─ [Activity] MainActivity 创建
    │  完整类名: com.example.app.ui.MainActivity

###  2.3 获取当前前台 Activity

有时候你只想知道当前显示的是哪个页面，不需要追踪完整的生命周期：


    // get_current_activity.js
    // 获取当前前台 Activity

    function getCurrentActivity() {
        var result = null;
        Java.perform(function() {
            var ActivityThread = Java.use("android.app.ActivityThread");
            var activities = ActivityThread.currentActivityThread()
                .mActivities.value;

            var iterator = activities.values().iterator();
            while (iterator.hasNext()) {
                var activityRecord = iterator.next();
                // ActivityClientRecord 中的 paused 字段表示是否在前台
                var isPaused = activityRecord.paused.value;
                if (!isPaused) {
                    var activity = activityRecord.activity.value;
                    result = activity.getClass().getName();
                    break;
                }
            }
        });
        return result;
    }

    // 使用
    // console.log("当前页面: " + getCurrentActivity());


> **Android 14+ hiddenapi 注意** ： ` ActivityThread.mActivities ` 与 `
> ActivityClientRecord ` 字段在 Android 9+ 进入 non-SDK 限制名单，Android 14
> 起进一步收紧。Frida 已能自动绕过限制，但 logcat 中可能看到 ` Accessing hidden field ` 警告 —— 不影响读值。

##  三、Hook Intent：拦截页面间的数据传递

###  3.1 Intent 的角色

在 Android 中，组件之间（Activity ↔ Activity, Activity ↔ Service, Activity ↔
BroadcastReceiver）的通信都通过 **Intent** 进行。Intent 携带了目标组件的信息和要传递的数据（Extras）。

Hook Intent 可以捕获：

  * 页面跳转时传递的用户 ID、订单号、Token
  * 启动 Service 时传递的任务参数
  * 发送广播时携带的数据

###  3.2 Hook startActivity 和 sendBroadcast

要 hook "所有的 startActivity 调用"，新手往往直接挂在 ` Activity.startActivity ` 上 ——
这其实漏了一大片。所有入口在底层都汇聚到 ` Instrumentation.execStartActivity ` 这一个方法，hook
在这里才是最稳的"漏斗底部"：

![Intent
Intent 启动调用链收敛图


    // intent_monitor.js
    // 监控所有 Intent 发送

    Java.perform(function() {
        var Activity = Java.use("android.app.Activity");
        var ContextWrapper = Java.use("android.content.ContextWrapper");

        // ====== 工具函数：格式化打印 Intent ======
        function printIntent(tag, intent) {
            if (intent === null) return;

            var action = intent.getAction();
            var component = intent.getComponent();
            var data = intent.getData();
            var flags = intent.getFlags();
            var categories = intent.getCategories();

            console.log("\n[" + tag + "]");

            if (component !== null) {
                console.log("  目标: " + component.getClassName());
            }
            if (action !== null) {
                console.log("  Action: " + action);
            }
            if (data !== null) {
                console.log("  Data: " + data.toString());
            }
            if (flags !== 0) {
                console.log("  Flags: 0x" + flags.toString(16));
            }

            // 打印 Extras（最重要的部分）
            var extras = intent.getExtras();
            if (extras !== null) {
                console.log("  Extras:");
                var keys = extras.keySet();
                var iter = keys.iterator();
                while (iter.hasNext()) {
                    var key = iter.next().toString();
                    try {
                        var value = extras.get(key);
                        if (value !== null) {
                            var valueStr = value.toString();
                            var valueType = value.getClass().getSimpleName();

                            // 对长值截断
                            if (valueStr.length > 300) {
                                valueStr = valueStr.substring(0, 300) + "...(" + valueStr.length + " chars)";
                            }

                            console.log("    " + key + " [" + valueType + "] = " + valueStr);
                        }
                    } catch(e) {
                        console.log("    " + key + " = (读取失败)");
                    }
                }
            }
        }

        // ====== Hook Instrumentation.execStartActivity（所有 startActivity 的汇聚点）======
        // 比 hook Activity.startActivity 更稳：覆盖所有 overload、所有上下文，
        // 也包括 AndroidX ActivityResultLauncher 走的 ActivityResultRegistry 路径。
        try {
            var Instrumentation = Java.use("android.app.Instrumentation");
            Instrumentation.execStartActivity.overloads.forEach(function(overload) {
                overload.implementation = function() {
                    // 参数表最后一个 Intent 之前的位置——直接遍历找
                    for (var i = 0; i < arguments.length; i++) {
                        if (arguments[i] && arguments[i].$className === "android.content.Intent") {
                            printIntent("execStartActivity", arguments[i]);
                            break;
                        }
                    }
                    return overload.apply(this, arguments);
                };
            });
            console.log("[OK] startActivity 监控 (全 overload)");
        } catch(e) {
            console.log("[--] Instrumentation hook: " + e.message);
        }

        // ====== Hook startService（兼容 startForegroundService）======
        ["startService", "startForegroundService"].forEach(function(name) {
            try {
                ContextWrapper[name].overloads.forEach(function(overload) {
                    overload.implementation = function(intent) {
                        printIntent(name, intent);
                        return overload.call(this, intent);
                    };
                });
                console.log("[OK] " + name + " 监控");
            } catch(e) {}
        });

        // ====== Hook sendBroadcast / sendOrderedBroadcast ======
        ["sendBroadcast", "sendOrderedBroadcast", "sendStickyBroadcast"].forEach(function(name) {
            try {
                ContextWrapper[name].overloads.forEach(function(overload) {
                    overload.implementation = function() {
                        printIntent(name, arguments[0]);
                        return overload.apply(this, arguments);
                    };
                });
                console.log("[OK] " + name + " 监控");
            } catch(e) {}
        });
    });


> **安全分析价值** ：很多 App 在 Intent Extras 中传递敏感数据（明文 Token、用户信息、支付金额），这些数据如果被恶意 App
> 截获（通过 BroadcastReceiver 或 exported Activity），就构成了安全漏洞。通过这个脚本可以快速发现这类问题。

###  3.3 Hook BroadcastReceiver 接收侧

发送侧（ ` sendBroadcast ` ）已在 §3.2 覆盖；接收侧才是"App 收到了什么 → 触发了什么动作"的关键。常见思路是遍历所有 `
BroadcastReceiver ` 子类挂 ` onReceive ` ——但这漏了运行时注册的接收器、并且类加载时机难控。更稳的漏斗点是 `
android.app.LoadedApk$ReceiverDispatcher$Args.run() `
，所有静态/动态注册的接收器最终都在这里被分发执行：


    // broadcast_receive_monitor.js
    // 接收侧漏斗 hook：覆盖静态注册 + 动态注册 + 已运行时 ClassLoader 加载的接收器

    Java.perform(function() {
        var Args = Java.use("android.app.LoadedApk$ReceiverDispatcher$Args");

        Args.run.implementation = function() {
            try {
                // mCurIntent 在 API 21-34 的 AOSP 实现里字段名稳定；若 ROM 改名可加 fallback
                var intent = this.mCurIntent.value;

                // 通过外部类引用拿到 ReceiverDispatcher，再读 mReceiver 字段
                var dispatcher = this.this$0.value;
                var receiver = dispatcher.mReceiver.value;
                var receiverName = receiver !== null
                    ? receiver.getClass().getName()
                    : "(unknown)";

                console.log("\n[BroadcastReceive → " + receiverName + "]");
                if (intent !== null) {
                    var action = intent.getAction();
                    if (action) console.log("  Action: " + action);

                    var extras = intent.getExtras();
                    if (extras !== null) {
                        var keys = extras.keySet();
                        var iter = keys.iterator();
                        while (iter.hasNext()) {
                            var key = iter.next().toString();
                            var value = extras.get(key);
                            if (value !== null) {
                                var s = value.toString();
                                if (s.length > 200) s = s.substring(0, 200) + "...";
                                console.log("  Extra: " + key + " = " + s);
                            }
                        }
                    }
                }
            } catch(e) {
                console.log("[BR] hook 异常: " + e.message);
            }
            return this.run();
        };

        console.log("[OK] BroadcastReceiver 接收侧监控 (Args.run 漏斗)");
    });


> **字段稳定性** ： ` mCurIntent ` 与 ` mReceiver ` 在 Android 5-14 的 AOSP
> 中字段名一致，但部分深度定制 ROM（一些国产厂商）可能改名。若读字段抛 ` NoSuchFieldError ` ，回退到「枚举 `
> BroadcastReceiver ` 子类 + 挂 ` onReceive ` 」的笨办法：可以在 ` Java.perform ` 内 `
> Java.enumerateLoadedClasses({ onMatch: function(name) { if (...)
> hookOnReceive(name); }, onComplete: ... }) ` ，但要注意类还没加载就不会被命中——需要在每次新
> ClassLoader 出现时重跑。

运行效果：


    [BroadcastReceive → com.example.app.push.PushNotificationReceiver]
      Action: com.example.app.ACTION_PUSH_MESSAGE
      Extra: msg_id = 998877
      Extra: payload = {"type":"order","order_id":12345,"action":"refresh"}

    [BroadcastReceive → androidx.work.impl.background.systemalarm.ConstraintProxy$NetworkStateProxy]
      Action: android.net.conn.CONNECTIVITY_CHANGE


结合发送侧的 ` sendBroadcast ` 监控，就拿到了广播的完整闭环：谁发的（包名/Action）→ 谁收的（接收器类名）→
带了什么数据。这对分析推送链路、风控触发条件、跨组件协作都非常有用。

##  四、Hook SharedPreferences：监控本地配置读写

###  4.1 SharedPreferences 的作用

SharedPreferences 是 Android 最常用的轻量级持久化存储方案，存储在 ` /data/data/包名/shared_prefs/ `
下的 XML 文件中。Token、设备指纹、功能开关、首次启动标记几乎都落在这里——监控价值极高。

###  4.2 全面监控 SharedPreferences 读写


    // sharedprefs_monitor.js
    // 监控所有 SharedPreferences 的读写操作

    Java.perform(function() {

        // ====== 监控获取 SharedPreferences 实例 ======
        var Context = Java.use("android.content.Context");
        Context.getSharedPreferences.overload("java.lang.String", "int").implementation = function(name, mode) {
            console.log("[SP] 打开文件: " + name + ".xml (mode=" + mode + ")");
            return this.getSharedPreferences(name, mode);
        };

        // ====== 监控读取操作 ======
        var SharedPreferencesImpl = Java.use("android.app.SharedPreferencesImpl");

        SharedPreferencesImpl.getString.implementation = function(key, defValue) {
            var value = this.getString(key, defValue);
            // 只打印非空值和感兴趣的 key
            if (value !== null && value.toString().length > 0) {
                console.log("[SP GET] " + key + " = \"" + value + "\"");
            }
            return value;
        };

        SharedPreferencesImpl.getInt.implementation = function(key, defValue) {
            var value = this.getInt(key, defValue);
            if (value !== defValue) {
                console.log("[SP GET] " + key + " = " + value + " (int)");
            }
            return value;
        };

        SharedPreferencesImpl.getBoolean.implementation = function(key, defValue) {
            var value = this.getBoolean(key, defValue);
            console.log("[SP GET] " + key + " = " + value + " (bool)");
            return value;
        };

        SharedPreferencesImpl.getLong.implementation = function(key, defValue) {
            var value = this.getLong(key, defValue);
            if (value !== defValue) {
                console.log("[SP GET] " + key + " = " + value + " (long)");
            }
            return value;
        };

        console.log("[OK] SharedPreferences 读取监控");

        // ====== 监控写入操作 ======
        var Editor = Java.use("android.app.SharedPreferencesImpl$EditorImpl");

        Editor.putString.implementation = function(key, value) {
            console.log("[SP PUT] " + key + " = \"" + value + "\" (string)");

            // 高亮敏感字段
            var lowerKey = key.toLowerCase();
            if (lowerKey.indexOf("token") !== -1 || lowerKey.indexOf("session") !== -1 ||
                lowerKey.indexOf("password") !== -1 || lowerKey.indexOf("secret") !== -1 ||
                lowerKey.indexOf("auth") !== -1 || lowerKey.indexOf("cookie") !== -1) {
                console.log("   敏感数据写入！");
            }

            return this.putString(key, value);
        };

        Editor.putInt.implementation = function(key, value) {
            console.log("[SP PUT] " + key + " = " + value + " (int)");
            return this.putInt(key, value);
        };

        Editor.putBoolean.implementation = function(key, value) {
            console.log("[SP PUT] " + key + " = " + value + " (bool)");
            return this.putBoolean(key, value);
        };

        Editor.putLong.implementation = function(key, value) {
            console.log("[SP PUT] " + key + " = " + value + " (long)");
            return this.putLong(key, value);
        };

        Editor.remove.implementation = function(key) {
            console.log("[SP DEL] " + key);
            return this.remove(key);
        };

        Editor.clear.implementation = function() {
            console.log("[SP CLEAR] 清除所有数据！");
            return this.clear();
        };

        // ====== 提交标记 ======
        // 仅 putXxx 不会真正落盘；要在 commit/apply 之后才生效。
        // 加这两条 hook 才能分清"写了但没提交"与"已生效"。
        Editor.apply.implementation = function() {
            console.log("[SP ↻] apply()");
            return this.apply();
        };
        Editor.commit.implementation = function() {
            var ok = this.commit();
            console.log("[SP ↻] commit() = " + ok);
            return ok;
        };

        console.log("[OK] SharedPreferences 写入监控");
    });


> **覆盖范围与盲区** ：上面这套 hook 只拦截系统默认实现 ` android.app.SharedPreferencesImpl ` 。如果
> App 使用了下面这些方案，需要单独 hook 对应入口：
>
>   * **MMKV** （腾讯）：hook ` com.tencent.mmkv.MMKV ` 的 ` encode* ` / ` decode* `
> 方法
>   * **Jetpack DataStore** ：基于 Protobuf + Coroutine，hook `
> androidx.datastore.* ` 序列化层
>   * **EncryptedSharedPreferences** （Jetpack Security）：是 SP 的装饰器，本套 hook
> 落到的是密文。要拿明文，挂公开方法 `
> androidx.security.crypto.EncryptedSharedPreferences$Editor.putString /
> putInt / putBoolean / ... ` —— 进入这一层时参数还是明文，序列化与加密发生在内部私有 `
> putEncryptedObject(String, byte[]) ` ，想看序列化中间态可以再挂这个私有方法
>   * **多进程 SP** ：部分 ROM 走 ` MultiProcessSharedPreferences ` ，类名不同
>

上面四种实现里，前三种都走 ` SharedPreferences ` 接口（一个基础实现 + 两个装饰器），最终落到 `
SharedPreferencesImpl ` ；后两种 MMKV / DataStore 是「平行替代系」——完全独立于 SP
接口，本节脚本一次都不会命中。下图把 5 种实现的 hook 入口和能拿到的明文位置画在同一张图里，配上拿到 App 后的诊断顺序：

    // 监控 WebView 的所有行为

            };
            console.log("[OK] addJavascriptInterface 监控");
        } catch(e) {}

###  5.3 Hook JSBridge 方法调用

当你通过 ` addJavascriptInterface ` 的 Hook 发现了 Bridge 类名后，可以进一步 Hook 它的每个方法：


    // jsbridge_hook.js
    // Hook 特定的 JSBridge 实现类

    Java.perform(function() {
        // 假设通过上面的脚本发现 Bridge 类是 com.example.app.bridge.NativeBridge
        var bridgeClassName = "com.example.app.bridge.NativeBridge";

        try {
            var BridgeClass = Java.use(bridgeClassName);
            var methods = BridgeClass.class.getDeclaredMethods();

            for (var i = 0; i < methods.length; i++) {
                var methodName = methods[i].getName();
                var paramCount = methods[i].getParameterTypes().length;

                // Hook 每个有 @JavascriptInterface 注解的方法
                (function(name) {
                    try {
                        var overloads = BridgeClass[name].overloads;
                        overloads.forEach(function(overload) {
                            overload.implementation = function() {
                                var args = [].slice.call(arguments);
                                var argsStr = args.map(function(a) {
                                    return a !== null ? a.toString().substring(0, 300) : "null";
                                }).join(", ");

                                console.log("\n[JSBridge] " + name + "(" + argsStr + ")");

                                var result = this[name].apply(this, args);
                                if (result !== undefined && result !== null) {
                                    console.log("  返回: " + result.toString().substring(0, 500));
                                }
                                return result;
                            };
                        });
                    } catch(e) {}
                })(methodName);
            }

            console.log("[OK] JSBridge " + bridgeClassName + " 全方法 Hook");
        } catch(e) {
            console.log("[--] JSBridge: " + e.message);
        }
    });


##  六、Hook ContentProvider：监控数据库操作

###  6.1 ContentProvider 的用途

ContentProvider 是 Android 四大组件之一，用于在 App 内部或跨 App 共享结构化数据。App 内部常用它封装 SQLite
数据库操作。


    // contentprovider_monitor.js
    // 监控 ContentProvider 的 query/insert/update/delete 操作

    Java.perform(function() {
        var ContentResolver = Java.use("android.content.ContentResolver");

        // ====== query 操作（兼容 5/6 参 overload，API 26+ 新签名也覆盖）======
        // 5 参：Uri, projection[], selection, selectionArgs[], sortOrder
        // 6 参：Uri, projection[], selection, selectionArgs[], sortOrder, CancellationSignal
        // 4 参 (API 26+)：Uri, projection[], Bundle queryArgs, CancellationSignal —— Room/Jetpack 走这条
        ContentResolver.query.overloads.forEach(function(overload) {
            overload.implementation = function() {
                var uri = arguments[0];
                console.log("\n[ContentProvider] QUERY (" + overload.argumentTypes.length + " args)");
                console.log("  URI: " + (uri ? uri.toString() : "null"));

                // 5/6 参 overload 的 selection 在 [2]
                if (overload.argumentTypes.length >= 5) {
                    if (arguments[2]) console.log("  WHERE: " + arguments[2]);
                    if (arguments[3]) {
                        console.log("  Args: " + Java.use("java.util.Arrays").toString(arguments[3]));
                    }
                } else if (arguments[2]) {
                    // 4 参 overload：第 3 个参数是 Bundle queryArgs
                    console.log("  QueryArgs Bundle: " + arguments[2].toString());
                }

                var cursor = overload.apply(this, arguments);
                if (cursor !== null) {
                    console.log("  结果: " + cursor.getCount() + " 行");
                }
                return cursor;
            };
        });

        // ====== insert 操作 ======
        ContentResolver.insert.implementation = function(uri, values) {
            console.log("\n[ContentProvider] INSERT");
            console.log("  URI: " + uri.toString());
            if (values !== null) {
                console.log("  Values: " + values.toString().substring(0, 500));
            }
            return this.insert(uri, values);
        };

        // ====== update 操作 ======
        ContentResolver.update.implementation = function(uri, values, where, selectionArgs) {
            console.log("\n[ContentProvider] UPDATE");
            console.log("  URI: " + uri.toString());
            if (where) console.log("  WHERE: " + where);
            if (values) console.log("  Values: " + values.toString().substring(0, 500));
            return this.update(uri, values, where, selectionArgs);
        };

        // ====== delete 操作 ======
        ContentResolver.delete.implementation = function(uri, where, selectionArgs) {
            console.log("\n[ContentProvider] DELETE");
            console.log("  URI: " + uri.toString());
            if (where) console.log("  WHERE: " + where);
            return this.delete(uri, where, selectionArgs);
        };

        console.log("[OK] ContentProvider 操作监控");
    });

> getInstalledPackages ` / ` getPackageInfo ` 时，默认只返回 ` <queries> `
> 清单里声明过的包（或自身），未声明的查询会返回空或抛 ` NameNotFoundException ` 。所以"App 检查
> Magisk"这种行为可能根本不走 ` getPackageInfo ` ，而是直接 ` new
> File("/sbin/.magisk").exists() ` 或 hook ` Runtime.exec("pm list packages") `
> 。监控覆盖面要相应扩大。

##  八、实战：综合系统监控脚本

把上面所有 Hook 模块合并为一个综合脚本，加载后可以快速获取 App 的行为全景图：


    // system_monitor.js
    // 综合系统级监控脚本
    // 用法: frida -U -f com.example.app -l system_monitor.js --no-pause

    // ====== 配置：开关各个监控模块 ======
    var CONFIG = {
        activity: true,         // Activity 生命周期
        intent: true,           // Intent 数据
        broadcastReceive: true, // BroadcastReceiver 接收侧（漏斗点）
        sharedPrefs: true,      // SharedPreferences 读写
        webView: true,          // WebView 加载和 JSBridge
        contentProvider: false,  // ContentProvider（通常输出太多）
        clipboard: true,        // 剪贴板
        packageManager: true,   // 安装包查询
        toast: true             // Toast 提示
    };

    console.log("╔══════════════════════════════════════════╗");
    console.log("║  Android 系统级监控 v1.0                  ║");
    console.log("╚══════════════════════════════════════════╝\n");

    Java.perform(function() {
        // ====== Activity 监控 ======
        if (CONFIG.activity) {
            var Activity = Java.use("android.app.Activity");
            Activity.onCreate.overload("android.os.Bundle").implementation = function(bundle) {
                var name = this.getClass().getName().split(".").pop();
                console.log("[Activity] 创建 → " + name);

                var intent = this.getIntent();
                if (intent !== null) {
                    var extras = intent.getExtras();
                    if (extras !== null && extras.size() > 0) {
                        var iter = extras.keySet().iterator();
                        while (iter.hasNext()) {
                            var key = iter.next().toString();
                            var val = extras.get(key);
                            if (val !== null) {
                                console.log("  " + key + " = " + val.toString().substring(0, 200));
                            }
                        }
                    }
                }
                this.onCreate(bundle);
            };

            Activity.onResume.implementation = function() {
                console.log("[→] " + this.getClass().getName().split(".").pop());
                this.onResume();
            };
            console.log("[OK] Activity");
        }

        // ====== BroadcastReceiver 接收侧监控 ======
        if (CONFIG.broadcastReceive) {
            try {
                var Args = Java.use("android.app.LoadedApk$ReceiverDispatcher$Args");
                Args.run.implementation = function() {
                    try {
                        var intent = this.mCurIntent.value;
                        var receiver = this.this$0.value.mReceiver.value;
                        var name = receiver !== null
                            ? receiver.getClass().getName().split(".").pop()
                            : "?";
                        var action = (intent && intent.getAction()) || "(no action)";
                        console.log("[BR→] " + name + " : " + action);
                    } catch(_) {}
                    return this.run();
                };
                console.log("[OK] BroadcastReceive");
            } catch(e) {
                console.log("[--] BroadcastReceive: " + e.message);
            }
        }

        // ====== SharedPreferences 监控 ======
        if (CONFIG.sharedPrefs) {
            var SPImpl = Java.use("android.app.SharedPreferencesImpl");

            SPImpl.getString.implementation = function(key, def) {
                var val = this.getString(key, def);
                if (val !== null && val.toString().length > 0 && val !== def) {
                    console.log("[SP] " + key + " = \"" + val.toString().substring(0, 150) + "\"");
                }
                return val;
            };

            var Editor = Java.use("android.app.SharedPreferencesImpl$EditorImpl");
            Editor.putString.implementation = function(key, value) {
                console.log("[SP] " + key + " = \"" + (value || "null").substring(0, 150) + "\"");
                return this.putString(key, value);
            };
            console.log("[OK] SharedPreferences");
        }

        // ====== WebView 监控 ======
        if (CONFIG.webView) {
            var WebView = Java.use("android.webkit.WebView");
            WebView.loadUrl.overload("java.lang.String").implementation = function(url) {
                if (url.indexOf("javascript:") === 0) {
                    console.log("[WebView] JS注入: " + url.substring(11, Math.min(url.length, 200)));
                } else {
                    console.log("[WebView] 加载: " + url);
                }
                this.loadUrl(url);
            };

            try {
                WebView.addJavascriptInterface.implementation = function(obj, name) {
                    console.log("[WebView] JSBridge 注册: window." + name + " (" + obj.getClass().getName() + ")");
                    this.addJavascriptInterface(obj, name);
                };
            } catch(e) {}

            try {
                WebView.evaluateJavascript.implementation = function(script, cb) {
                    console.log("[WebView] eval: " + script.substring(0, 200));
                    this.evaluateJavascript(script, cb);
                };
            } catch(e) {}
            console.log("[OK] WebView");
        }

        // ====== PackageManager 监控 ======
        if (CONFIG.packageManager) {
            var PM = Java.use("android.app.ApplicationPackageManager");
            PM.getPackageInfo.overload("java.lang.String", "int").implementation = function(name, flags) {
                console.log("[PM] 查询: " + name);
                return this.getPackageInfo(name, flags);
            };
            console.log("[OK] PackageManager");
        }

        // ====== Clipboard 监控 ======
        if (CONFIG.clipboard) {
            try {
                var CM = Java.use("android.content.ClipboardManager");
                CM.setPrimaryClip.implementation = function(clip) {
                    var text = clip.getItemAt(0).getText();
                    console.log("[复制] " + (text !== null ? text.toString() : "(非文本)"));
                    this.setPrimaryClip(clip);
                };
                console.log("[OK] Clipboard");
            } catch(e) {}
        }

        // ====== Toast 监控 ======
        if (CONFIG.toast) {
            var Toast = Java.use("android.widget.Toast");
            Toast.makeText.overload("android.content.Context", "java.lang.CharSequence", "int")
                .implementation = function(ctx, text, duration) {
                console.log("[Toast] " + text);
                return this.makeText(ctx, text, duration);
            };
            console.log("[OK] Toast");
        }

        console.log("\n[*] 系统监控已就绪，等待 App 操作...\n");
    });

##  总结

本篇的核心价值是： **不需要了解 App 的业务代码，通过 Hook 系统 API 就能获得 App 行为的全景图。**

面对一个陌生 App，第一步永远是 §8 的综合监控脚本——加载它，操作 App，从输出里捞 Activity 跳转路径、SP 中的 Token
key、WebView 加载的 H5、JSBridge 类名、Intent Extras 里的敏感字段。这些信息合在一起就是"该往哪里写定向
Hook"的方向盘。

需要注意三个盲区：① SP hook 只覆盖系统默认实现，MMKV/DataStore/加密 SP 要单独挂；② Package Visibility 让
Android 11+ 的 ` getInstalledPackages ` 默认裁剪；③ 主流加固/混淆方案可能改写 Toast、自定义 SP
类名，遇到打不出日志的情况优先回到 ` Java.deoptimizeEverything() ` 与 `
Java.enumerateLoadedClasses ` 排查。
