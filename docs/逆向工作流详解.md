# 逆向工作流详解

> AGENTS.md / CLAUDE.md 中有工作流速查摘要。本文档是完整版本。

## 渐进式披露流水线

```
目标分类（文件类型/平台/架构）
  ↓
最小侦察（字符串/导入/manifest/脚本列表）         ← L0 轻量
  ↓
分类决策（portable / context-aware / runtime-assisted / triage-only）
  ↓
定向深挖（反编译/xref/Hook — 仅在确认需要后）     ← L1-L3 逐级
  ↓
产出汇总（report.md + findings.json + triage.md）
  ↓
审查门（所有claim有证据？triage已标注？敏感数据已脱敏？）
```

**核心规则**：不得跳阶段。

## 四个分析深度等级

| 等级 | 名称 | 何时使用 | 典型产出 |
|------|------|---------|---------|
| **L1** | 便携提取 | 纯算法、无环境依赖、无混淆 | 完整 Python/JS 复现脚本 |
| **L2** | 上下文感知 | 需要部分运行时值（nonce、时间戳、设备信息） | 带桩复现 + 环境缺口说明 |
| **L3** | 运行时辅助 | JNI/native 调用、动态代码加载 | Hook 脚本 + 调用链文档 |
| **L4** | Triage-only | WASM、VM 混淆、重度反调试、硬件绑定 | 阻滞点清单 + Hook 候选 + 建议 |

**不得假装**：L4 目标不要声称"已完成完整分析"。

## 目标分类与 skill 路由

| 特征 | 判定 | 主 skill |
|------|------|---------|
| `.apk` / `.aab` | Android APK | `apk-reverse` |
| `.exe` / `.dll` / `.sys` | Windows PE | `ida-reverse` |
| `.so` / `.elf` / `.macho` | Linux/macOS 二进制 | `ida-reverse` 或 `radare2` |
| `.js` / 网页 URL / 小程序 | Web JS | 默认 `ruyi-reverse`；仅需 CDP 完整断点且无反检测需求时用 `mcp-js-reverse-playbook` |
| 任意二进制（仅需快速侦察） | 通用 | `radare2` |

混合目标（APK 含 `.so`）先走 `apk-reverse` 侦察：仅做纯静态算法/结构恢复时切 `ida-reverse` / `radare2`；遇到反调试、反 Frida、syscall、匿名 RX、壳、VMP 或 Dex2C 时切 `native-reverse`。Web JS 目标默认从 `ruyi-reverse` 进入，利用 ruyi-mcp 的反检测、Observe、Capture、Trace、Human-Sim、Debug、Export 能力做统一侦察；只有明确需要 Chrome/CDP 级暂停、单步、作用域查看，且目标无强反检测时，才切到 `mcp-js-reverse-playbook` / `js-reverse-mcp`。

## APK 主链

```text
fingerprint 多 marker（framework + protector + DEX + ABI）
  -> decode.ps1（jadx + apktool + manifest summary）
  -> 主战场决策（Java/Kotlin | smali | runtime DEX | native | Unity/Flutter/RN）
  -> 需要时初始化 LDPlayer/Frida
  -> 标准 whole-DEX：dump-dex.ps1 / panda
  -> Kotlin 映射、API 候选、定向 Hook 或 patch
  -> rebuild-sign-install.ps1
  -> 三件套 + workspace.json 审查门
```

框架 marker 不是互斥路由。企业壳/hybrid APK 可以同时包含 Flutter runtime、多个 DEX、Java 业务代码和壳 `.so`；不得因为单一 `libflutter.so` 就停止 DEX/Manifest 分析。

LDPlayer + panda 路线已经验证可用于运行时存在标准完整 DEX 的 whole-DEX 恢复，包括一个 360 Jiagu VIP 样本的部分业务 DEX。`complete-enough` 只代表结构校验通过，不代表完整脱壳；方法抽取、CDEX、VMP、Dex2C 和强 anti-dump 另行分流。

## 分类决策 Marker

| Marker | 倾向深度 |
|--------|---------|
| 纯 Java/Kotlin，无 native 调用 | L1 或 L2 |
| `System.loadLibrary()`，.so 占比 < 30% | L2（Java）+ L3（native） |
| 核心逻辑在 .so 中 | L2（Java）+ L3（native） |
| `.wasm` / `WebAssembly.instantiate` | **L4** |
| VM 解释器（opcode switch/while loop） | **L4** |
| 控制流平坦化 / 字符串加密 / 反调试 | **L4** |
| 反 Hook / 反模拟器 / 环境检测 | L3（需先中和保护） |

## 交付产物规范

每次完整分析产出**三件套**（模板在 `skill/reverse-coordinator/templates/`）：

| 产物 | 模板 | 内容 |
|------|------|------|
| `report.md` | `templates/report.md` | 人类可读报告：基本信息、关键发现、架构概览、敏感点清单 |
| `findings.json` | `templates/findings.json` | 结构化发现：每条有 `location`/`evidence`/`confidence`/`rebuild_status` |
| `triage.md` | `templates/triage.md` | 未完成项：阻滞点、环境缺口、待验证假设、未来方向 |

辅助：`workspace.json`（`templates/workspace.json`）记录会话阶段和审查门状态。

### findings.json 条目规范

```json
{
  "id": "F-001",
  "category": "signature|crypto|network|anti-tamper|jni|logic|config|other",
  "claim": "签名函数位于 XxxClass",
  "location": { "file": "", "class": "", "method": "", "address": "" },
  "evidence": {
    "type": "string_xref|disasm|decompile|runtime_hook|network_trace|manifest",
    "source": "jadx 反编译 com/example/LoginActivity.java",
    "value": "被 MainActivity.onCreate 调用"
  },
  "confidence": "high|medium|low"
}
```

## 审查门自检清单

产出交付前逐项确认：

- [ ] 所有 claim 都有 evidence 支撑（地址/xref/调用栈/运行时值）
- [ ] 没有"猜测性"结论混入 findings（confidence=low 的必须标注）
- [ ] Triage 遗留项已标注原因和下一步建议
- [ ] 复现脚本能独立运行（如有）
- [ ] 敏感数据（token/cookie/设备ID）已脱敏或标注为仅本地
- [ ] findings.json 中每条有 `confidence` 字段
- [ ] 没有对 L4 目标声称"已完整还原"

## 各 skill 的阶段映射

| 阶段 | apk-reverse | ida-reverse | ruyi-reverse / mcp-js-reverse-playbook | radare2 |
|------|------------|-------------|-----------------------------------------|---------|
| L0 分类 | `fingerprint.sh` 多 marker + manifest 摘要 | `rabin2 -I` | 页面 URL/框架/反检测信号判定 | `rabin2 -I` |
| L1 侦察 | `decode.ps1` + jadx/apktool 产物 | `survey_binary` | ruyi Observe+Capture；CDP 专项才用 js-reverse Observe | `rabin2 -z/-i` |
| 决策 | Java/smali/native 判定 | marker 判定 L1-L4 | WASM/VM/指纹检测/断点需求判定 | 复杂度判定 |
| L2 深挖 | smali patch | `analyze_function` | ruyi Trace+Export+Patch；CDP 单步需求切 js-reverse | `r2` 交互分析 |
| L3 深挖+ | Frida Hook / whole-DEX / native handoff | `xrefs_to`/`callgraph` | ruyi 反检测浏览+人类模拟；必要时 `ruyi_export_session` 桥接 js-reverse | `axt`/patch |
