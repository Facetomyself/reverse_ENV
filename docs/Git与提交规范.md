# Git 与提交规范

> 当前仓库已初始化 Git，主分支为 `main`；`workspace/` 下允许存在受 registry 管理的独立仓库。

---

## 仓库硬性规则

1. **禁止猜测**：任何结论必须基于真实的命令输出、真实的 diff、真实的代码上下文。
2. **开发基线**：新仓库默认分支为 `main`；已有子仓和外部上游仓保留其真实默认分支，不为统一命名强制改写历史。
3. **提交前对齐**：发起提交前先读取当前 branch 及其 upstream，再同步对应远端分支；不得把 `origin/main` 硬套到使用 `master` 或其他分支的子仓。
4. **脏改动处理**：若工作区存在无法确认归属的脏改动，必须先停下来告知，不得偷偷带入提交。
5. **不保留无用旧代码**：修改时若确认旧路径/旧引用无其他依赖，应一并清理，不为"看起来兼容"堆砌。
6. **提交信息规范**：必须描述真实功能结果。**禁止**使用 `update`、`fix bug`、`merge branch`、`修改一下` 这类空洞信息。

   正确示例：
   ```
   安装 radare2 6.1.8 便携版到 tools\radare2\
   更新 decode.ps1 中的 jadx/apktool fallback 路径
   新增 js-reverse-mcp 到 .mcp.json
   ```

7. **操作须确认**：不得假装完成过某项操作。所有工具安装、配置修改必须有真实输出作为证据。
8. **禁止越权**：未经明确授权，不得进行破坏性操作（删除技能文件、卸载工具、修改全局配置）。
9. **多仓隔离**：主仓提交不得夹带 workspace 子仓内容；子仓提交也不得修改主仓 gitlink 或 registry。
10. **活动项目保护**：标记为 `deferred-active` 的项目禁止执行 checkout/reset/clean/stash/rebase、目录移动和 Git 目录吸收操作。
11. **原始证据禁入 Git**：第三方二进制、抓包、IDA 数据库、Cookie、凭据、浏览器 profile、解包和反编译全集只留本地，不使用 Git LFS 规避此规则。
12. **远端默认权限**：新建 workspace 和知识库仓库默认为 Private；转为 Public 前必须完成版权、敏感信息和证据脱敏审查。
13. **跨仓提交顺序**：先在子仓完成验证、commit 和 push，再在主仓更新 gitlink/registry；不得先提交一个尚未推送、其他机器无法取得的 submodule commit。

## 标准执行顺序

### 阶段 1：环境确认

操作前先确认：
- 当前工作目录为 `D:\reverse_ENV\`
- 相关文件存在且可读写
- 若涉及 MCP，确认对应 Python/Node 环境可用
- 若涉及 workspace，读取 `docs/workspace-projects.yaml` 并确认目标不是 `deferred-active`

### 阶段 2：阅读上下文

改动前必须：
- 阅读相关 `skill/*/SKILL.md` 和脚本
- 阅读 `CLAUDE.md` 中的架构说明
- 确认改动不会破坏已有 MCP 配置

### 阶段 3：执行改动

- 先改核心文件（脚本、配置、skill 定义）
- 同步更新所有引用（CLAUDE.md、skill 文档）
- 清理旧引用、旧路径、旧用户名

### 阶段 4：自检

改动后必须：
- 验证新增/修改的脚本语法正确
- 验证新增工具版本号正确
- 验证 `.mcp.json` JSON 合法
- 验证 CLAUDE.md 中的路径均真实存在
- 运行 workspace audit，确认 registry、remote、submodule 和禁止文件检查通过
- 清理临时文件

### 阶段 5：提交（若已初始化 git）

- 再次确认工作区只包含本次改动文件
- 若涉及 submodule，执行 `git submodule status --recursive`，并分别检查主仓与目标子仓状态
- 提交信息描述真实改动
- 确认无敏感信息混入 diff

## 操作清单

每次操作完成后自检：

- [ ] 是否先阅读了相关 skill 和脚本？
- [ ] 所有路径是否指向 `D:\reverse_ENV\` 内真实文件？
- [ ] CLAUDE.md 是否已同步更新？
- [ ] `.mcp.json` 是否合法？
- [ ] 是否有旧用户名 `25286`、旧路径 `D:\APP\IDA` 残留？
- [ ] 新增文件是否有中文乱码？
- [ ] 是否清理了临时下载文件和缓存？
- [ ] diff 中是否混入了敏感信息？
