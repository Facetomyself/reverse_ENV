# 多源搜索编排规范

> CLAUDE.md / AGENTS.md §多源搜索编排（硬纪律）的详细版。渐进式披露——正文只放触发规则，细则在此。
>
> `search-layer` 是全局搜索分级策略，不是本仓库内的项目 MCP。Claude 全局环境已配置为 WebSearch + Exa + Tavily + Grok 并行；Codex 侧已迁移本地 skill 副本到 `~/.codex/skills/search-layer`，并已通过 `search.py --mode fast --intent resource` 的 Exa smoke test。

## 触发时机

| 时机 | 场景 | 动作 |
|------|------|------|
| **时机 1** | 新项目、新目标、新分析任务启动 | 先搜，确认无现成方案再进 IDA/jadx/Frida |
| **时机 2** | 分析阻塞（算法矛盾、加固失败、未知协议） | 暂停当前路径，搜替代方案 |

## 执行流程

```
触发 → search-layer (WebSearch+Exa+Tavily+Grok 并行)
     → GitHub search_code (类名/函数名/包名/唯一字符串)
     → 有现成? → 评估质量 → 复用/参考
     → 无 → 确认 → 进入最小侦察
```

## 环境分层

| 环境 | search-layer 状态 | 要求 |
|------|-------------------|------|
| Claude | 全局 MCP 已配置 | 直接按本规范调用 |
| Codex | 已迁移本地 skill 副本，fast 模式已验证 | 不写入项目 `.mcp.json`；本地路径为 `~/.codex/skills/search-layer`，后续按需继续验证 Tavily/Grok 深度模式 |
| 项目仓库 | 只记录使用规范 | 不保存 API Key，不复制全局密钥，不把搜索 MCP 当作项目冷启动服务 |

不能把“仓库里没有 search-layer”判定为项目缺口；它是用户级 skill/搜索分级能力，不进入项目 `.mcp.json`。若当前会话未加载该 skill，应标注为“Codex skill 未加载/需重启会话”，再按可用的官方搜索、GitHub 或浏览器方案 fallback。

## 关键词构造

1. 中英文变体各 ≥2 组
2. 覆盖：`目标名+sign` / `+crack` / `+decrypt` / `+reverse` / `+api`
3. GitHub `search_code`：核心类名、native 函数名、唯一字符串、作者名

## 自查信号

以下任一信号出现 → 立即触发时机 2：

- 连续 3 次同一 SO/函数试错
- 分析结论与抓包数据矛盾（如 hex vs binary）
- 未经搜索直接打开反编译工具
- 以「可能没有现成方案」为由跳过搜索
