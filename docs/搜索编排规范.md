# 多源搜索编排规范

> CLAUDE.md §多源搜索编排（硬纪律）的详细版。渐进式披露——正文只放触发规则，细则在此。

## 触发时机

| 时机 | 场景 | 动作 |
|------|------|------|
| **时机 1** | 新项目、新目标、新分析任务启动 | 先搜，确认无现成方案再进 IDA/jadx/Frida |
| **时机 2** | 分析阻塞（算法矛盾、加固失败、未知协议） | 暂停当前路径，搜替代方案 |

## 执行流程

```
触发 → search-layer (WebSearch+Exa+Tavily+Grok 并行)
     → GitHub search_code (类名/函数名/包名/唯一字符串)
     → 有现成? → 评估质量 → 复用/参考
     → 无 → 确认 → 进入最小侦察
```

## 关键词构造

1. 中英文变体各 ≥2 组
2. 覆盖：`目标名+sign` / `+crack` / `+decrypt` / `+reverse` / `+api`
3. GitHub `search_code`：核心类名、native 函数名、唯一字符串、作者名

## 自查信号

以下任一信号出现 → 立即触发时机 2：

- 连续 3 次同一 SO/函数试错
- 分析结论与抓包数据矛盾（如 hex vs binary）
- 未经搜索直接打开反编译工具
- 以「可能没有现成方案」为由跳过搜索
